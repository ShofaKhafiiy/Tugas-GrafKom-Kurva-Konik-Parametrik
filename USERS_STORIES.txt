================================================================================
  USER STORIES - Kurva Konik Parametrik
================================================================================

Project dibagi menjadi 4 User Story, masing-masing dikerjakan 1 anggota tim.

Urutan merge WAJIB:
  main -> US 1.1 -> US 1.2 -> US 2.1 -> US 2.2 -> main


================================================================================
[US 1.1] User - Memilih Bentuk Geometri Parametrik
================================================================================

Deskripsi:
Sebagai User, saya ingin dapat memilih salah satu dari 4 bentuk geometri
(Lingkaran, Elips, Parabola, Hiperbola) agar sistem dapat menyiapkan
ruang kerja dan parameter yang relevan.

🎯 Acceptance Criteria
- Sistem menyediakan elemen <select> untuk memilih jenis kurva.
- Saat bentuk dipilih, form input beradaptasi secara dinamis:
  Lingkaran (Xc, Yc, r, delta, tMin, tMax)
  Elips (Xc, Yc, a, b, delta, tMin, tMax)
  Parabola (Xc, Yc, a, delta, tMin, tMax)
  Hiperbola (Xc, Yc, a, b, delta, tMin, tMax)

🛠️ Technical Tasks
[Frontend] public/index.html: struktur UI, <select>, <div id="parameter-inputs">
[Frontend] public/js/ui.js: event listener dropdown, renderParams(), getParamDefs()
[Frontend] public/css/style.css: styling form


================================================================================
[US 1.2] User - Memanipulasi Parameter dan Kalkulasi Delta
================================================================================

Deskripsi:
Sebagai User, saya ingin memasukkan parameter pusat, konstanta kurva, dan
delta agar sistem menghitung koordinat titik dengan resolusi yang diinginkan.

🎯 Acceptance Criteria
- Input Xc, Yc, r/a/b, delta, tMin, tMax tersedia.
- Validasi: delta harus > 0.
- Sistem menghasilkan array [{x, y}, ...] sesuai rumus parametrik.

🛠️ Technical Tasks
[Logika] public/js/geometryCalc.js: 4 fungsi kalkulasi parametrik
[Frontend] public/js/ui.js: validate(), processCurve(), koneksi tombol


================================================================================
[US 2.1] System - Visualisasi Proses Render Berkelanjutan
================================================================================

Deskripsi:
Sebagai User, saya ingin melihat kurva digambar bertahap (titik demi titik)
di kanvas untuk memvisualisasikan proses looping.

🎯 Acceptance Criteria
- Gambar tidak muncul seketika, tapi step-by-step.
- Pakai requestAnimationFrame() untuk jeda stabil.
- Kanvas dibersihkan tiap kali tombol proses ditekan.
- Sistem koordinat: pixelX = w/2 + x, pixelY = h/2 - y

🛠️ Technical Tasks
[Frontend] public/index.html: elemen <canvas id="renderCanvas">
[Frontend] public/js/canvasAnimator.js: mapCoordinate(), drawGrid(), animateCurve()
[Frontend] public/css/style.css: styling canvas


================================================================================
[US 2.2] System - Visualisasi Bentuk Akhir & Live Tracking
================================================================================

Deskripsi:
Sebagai User, saya ingin memantau koordinat real-time dan melihat notifikasi
"Selesai" beserta informasi kurva.

🎯 Acceptance Criteria
- Panel "Koordinat saat ini: X=..., Y=..." selama animasi.
- Notifikasi "Selesai!" dengan waktu eksekusi dan total titik.
- Panel analisis menampilkan rumus dan properti kurva.
- (Opsional) POST /api/save-curve saat selesai.

🛠️ Technical Tasks
[Frontend] public/index.html: live-coordinates, render-status, analysis-panel
[Frontend] public/js/canvasAnimator.js: update DOM, fetch POST
[Frontend] public/js/ui.js: showAnalysis()
[Backend] server.js: Express + POST /api/save-curve
[Backend] package.json: dependency express
